# v0.8.0-alpha.15

Library Tool `0.8.0-alpha.15` is a desktop prerelease for hands-on testing.
Library Tool Capture remains a separately published Android product at
`0.5.1-alpha.7`; this desktop release does not rebuild or replace it.

## Highlights

- The framework-neutral Library Engine now owns substantially more of the
  catalogue, representation, canvas, Replica, translation, job, provider,
  interchange, and text-layer behavior behind stable contracts. The browser is
  a client of that engine instead of the long-term home for product rules.
- Replica can propose page regions automatically from its sparkle action or
  `Shift+D`. Detection is asynchronous and replay-safe, and a proposal remains
  separate from saved human work until it is explicitly applied.
- Provider credentials have moved out of ordinary project and browser state
  into the protected engine secret boundary, with account-scoped synchronization
  and migration of earlier local values.
- Capability discovery can now assemble only the modules and workbenches an
  installation supports. This is the executable foundation for a future
  KiCad-style manager with independently launchable Catalogue, Replica,
  Research, and Publishing workbenches.

## Changed

- Catalogue edits, PDF representation changes, canvas preparation, Replica
  proposals, translation pages, `.lib` interchange, and background work use
  explicit revisions, durable operation identities, or both where required.
- Replica preserves stable region identities and extensions, protects verified
  or human-edited pages from automatic replacement, and supports apply/dismiss
  review of machine proposals.
- Page-aligned translations are exposed through a provider-neutral engine API
  and can be selected as the Replica preview text layer. Translation generation
  remains a separately discoverable capability rather than a browser-specific
  dependency.
- Workbench and item availability are derived from installed capability
  contributions, allowing optional components to be absent without leaving
  unusable commands behind.
- Standalone maintenance tools take credentials only from explicit environment
  variables. Worktree seeding preserves useful preferences and work state while
  stripping every registered legacy credential field.
- The desktop shell now uses Electron 43 and Node.js 22.12 or newer. Release
  publication is draft-first, channel-strict, artifact-allowlisted, and safe to
  rerun.

## Fixed

- Automatic detection no longer overwrites protected page work before the user
  accepts its proposal, and detected figure assets are staged until acceptance.
- Detection retries and ambiguous responses reuse durable operation receipts
  instead of starting duplicate work, including after a process restart.
- Provider discovery fails closed when the engine does not advertise the
  required capability.
- Mistral credential replacement, clearing, migration, synchronization, account
  switching, and crash recovery no longer pass through the general settings
  payload.
- Packaged sidecar startup and desktop transport are authenticated and restricted
  to the expected local client boundary.

## Architecture direction

The accepted direction is a small project manager that opens independent
workbenches over the same library data. Modules contribute capabilities,
commands, readiness, and views; an installation can therefore omit Replica,
research/RAG, translation generation, or publishing without weakening the
remaining tools. The engine has no Flask or Electron dependency, so a later Qt
client can embed it directly and a Godot client can use the same local protocol.

The current browser Replica tab is still a transitional client. This release is
primarily the flexible engine, integrity, security, and release foundation for
the planned interface rebuild, not the final workbench redesign.

## Known caveats

- This remains a prerelease and should be tested on backed-up library data.
- The KiCad-style launcher and separately packaged workbenches are not yet
  implemented; all current modules still ship together.
- Translation viewing is engine-backed, but the streamlined translation-
  generation workflow planned for the independent Replica workbench is not yet
  complete.
- Windows installer signing depends on the configured release certificate. A
  privately managed certificate can still trigger SmartScreen on machines that
  do not trust its root.
