# v0.8.0-alpha.12

Library Tool `0.8.0-alpha.12` and Android Book Capture `0.5.1-alpha.4`
are prerelease builds for hands-on testing.

## Highlights

- Added Porcelain and Slate desktop themes with theme-specific fonts, activity
  icons, tab tints, and stronger contrast.
- Added a shared Remarks sidebar for attention items across Catalogs, Sources,
  and Entries, including editable reasons and navigation back to each record.
- Added Info > Resources, showing the availability, source, location, size,
  update time, and record count for local databases.
- Added optional facsimile paper tinting sampled from PDF margins. OCR
  comparison is now the default PDF view while explicit user choices remain
  respected.
- Expanded the Workbench with inline verification, clearer readiness and phase
  navigation, contextual page tools, improved OCR staging feedback, and easier
  entry creation.
- Android Book Capture adds explicit local and LAN capture, cloud claiming,
  Fast and More Detail camera profiles, continuous light, preview sharpening,
  camera diagnostics, a shallow capture queue, and clearer processing and
  delivery status.

## Changed

- Publishing saves the Workbench metadata currently on screen before checking
  readiness or opening a confirmation that summarizes the title, rights, and
  PDF being published.
- OCR staging opens the jobs drawer, names its target, reports completion more
  clearly, and supports Ctrl/Cmd+S in the text-editing phase.
- Android freezes capture ownership when a book starts. Local books remain
  local until a signed-in account explicitly claims them.
- Android processing and uploads are per-entry, recoverable, bounded, and
  idempotent, with truthful waiting, processing, partial, failed, uploaded, and
  imported states.
- LAN pairing requires a branded nonce response, and delivery receipts identify
  both the submitted capture and the generated desktop entry.
- Desktop and Android profile-secret synchronization uses revision-aware
  conflict handling so one device cannot silently erase newer keys from
  another.

## Fixed

- Prevented stale Workbench saves, verification callbacks, publish messages,
  publish completion, and network responses for one book from repainting,
  clearing, or publishing another selected book.
- Prevented failed whole-state writes from resurrecting stale Remarks data;
  writes are serialized and retried with backoff.
- Fixed LAN receipts conflating the capture ID with the generated entry ID.
- Fixed textless PDFs erasing the user's OCR comparison preference.
- Fixed Android discard and delete failures being reported as success or
  clearing an active capture before its queue copy was safely moved.
- Preserved stale-recognition protection while refining Android voice-command
  behavior.
- Kept Android processing resolution consistent with the selected camera
  profile and rejected incomplete deliveries instead of silently uploading
  partial books.
- Fixed hidden desktop controls becoming visible because of CSS specificity,
  including a destructive rollback action.
- Prevented mixed icon sets during rapid theme changes and corrected several
  theme contrast, control-height, and icon-state defects.
- Fixed APK signer verification for newer `apksigner` output.
- Restored the released cloud authorization policies and held back the
  unfinished member-approval experiment until its client and RLS design are
  complete.
- Removed direct API-role access to the trigger-only profile-secret revision
  function.

## Validation and infrastructure

- Added Android unit-test, lint, and debug-build checks to pull-request CI.
- Added behavior coverage for Remarks, PDF preferences, and cross-book
  Workbench save and publish races.
- The local release gate passed 575 Python tests, 82 JavaScript behavior tests,
  Ruff, Android unit tests, Android lint, and a clean debug APK build.

## Known caveats

- This remains a prerelease.
- The Android camera, voice, and LAN changes have automated coverage but still
  merit a real-device smoke test and capture-latency check.
- LAN delivery is limited to trusted private or local addresses and uses
  cleartext transport within that trusted network.
- Member roles and account approval remain intentionally deferred; normal
  released sign-in behavior is restored.
- Further corrections to already-delivered Android books are handled in the
  desktop Library Tool.
- The Windows installer is signed only when a release certificate is
  configured; otherwise Windows SmartScreen may warn.
