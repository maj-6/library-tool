# v0.8.0-alpha.13

Library Tool `0.8.0-alpha.13` and Android Book Capture `0.5.1-alpha.5`
are prerelease builds for hands-on testing.

## Highlights

- Added per-tab undo and redo so each workspace retains its own history across
  tab switches and asynchronous changes.
- Smart Scan now prefers an attached local PDF and can pause for
  keyboard-driven title-page marking before saving and scanning a compact page
  selection.
- Added consistent volume labels across Library Tool, processing jobs, and the
  public Archive Browser without changing stored titles.
- Extended Remarks to replica pages and published volumes, including shared
  replies, resolution, and navigation back to the exact item.
- Added contributor avatars and profile popups, and reframed Home around recent
  activity, review work, and published books.

## Changed

- Consolidated Catalog and Replica toolbars into responsive groups and menus
  that remain usable in narrower windows.
- Catalog views now distinguish empty collections from filtered results, report
  visible and total rows, and retain year filtering across both tables.
- Published-volume shortcuts open the Workbench only when one exact local entry
  matches the public slug.
- Removed unavailable Azure Document Intelligence and OpenAI Vision OCR
  choices. Legacy credentials remain local-only while settings normalize to a
  supported engine.
- Credential settings now report read and write failures and preserve visible
  values when the local service cannot be reached.
- Desktop startup and updater windows now follow Porcelain, Slate, and the base
  palette of custom themes.
- Updated project, deployment, workflow, and architecture documentation to
  match the current application and cloud boundaries.

## Fixed

- Kept page attention marks and discussion threads attached to the correct
  physical pages after deletion or renumbering. A committed deletion now
  reports any derivative write failure as partial success with a visible
  review warning and backup name.
- Preserved Remarks labels and categories when interface settings are reset.
- Restored native Space and Enter behavior on focused Smart Scan controls.
- Prevented stale page-review revisions from being applied after deletion.
- Kept shared popup menus onscreen at non-default interface zoom.
- Refreshed the published catalogue after publishing and distinguished an empty
  catalogue from one that is still loading.
- Fixed clipped or duplicated controls in narrow Workbench layouts.
- Improved keyboard navigation, focus restoration, accessible names, and
  selection state across icon controls, activity rows, book lists, phase
  controls, and replica tools.

## Validation and infrastructure

- CI now runs the Smart Scan, per-tab history, volume-title, Workbench source,
  and settings behavior suites.
- The local release gate passed Ruff, 623 Python tests, 120 JavaScript behavior
  tests, release metadata checks, and source syntax checks.
- A final packaged-desktop smoke test verified the alpha.13 shell, updater
  manifest hashes, and preservation of page Remarks through a settings reset.
- The Android gate passed 58 unit tests, lint with no errors, and a debug APK
  build for version `0.5.1-alpha.5` (`versionCode` 24).

## Known caveats

- This remains a prerelease.
- Manual Smart Scan page marking, zoomed menus, contributor profiles, and
  page/publication Remarks navigation still merit a packaged desktop smoke
  test.
- Manually selected Smart Scan documents process every marked page rather than
  the automatic scan limit.
- Windows installer signing depends on the configured release certificate. The
  current privately managed certificate may still trigger SmartScreen on
  machines that do not trust its root.
